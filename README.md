# Proyecto Urban Grocers

## Pruebas al back-end a aplicación de Plataforma de entrega de productos por catálogo y kits preparados

**Logros en el proyecto Urban Grocers:**

- Elaboré una **lista de comprobación (checklist) para validar una nueva funcionalidad del backend de la API**, asegurando que todos los endpoints cumplieran con los requisitos funcionales y de negocio.
- Realicé pruebas **manuales de API utilizando Postman**, enviando solicitudes `GET`, `POST`, `PUT` y `DELETE` con distintos cuerpos de datos (válidos e inválidos) para verificar:
    - Respuestas del servidor (códigos de estado HTTP)
    - Mensajes de error apropiados
    - Validación de campos requeridos
    - Integridad de datos en respuestas JSON
- Detecté y documenté **bugs y comportamientos inesperados** directamente en **Jira**, con descripciones detalladas, pasos para reproducirlos y evidencias adjuntas (capturas de pantalla o exportaciones de Postman).
- Verifiqué aspectos clave como:
    - Autenticación de usuarios
    - Accesos no autorizados
    - Manejo de productos en inventario desde la API
    - Lógica de actualización de datos (por ejemplo, stock y precios)
- Participé en el flujo de trabajo tipo Scrum.


### Herramientas usadas: Postman, JIRA

### Conclusiones – Proyecto Urban Grocers
El proyecto Urban Grocers, enfocado en las pruebas del backend de una plataforma de entrega de productos por catálogo y kits preparados, permitió validar de manera sistemática la estabilidad, confiabilidad y cumplimiento funcional de la API.
A partir del trabajo realizado, se concluye que:

1. Se garantizó la calidad funcional de la API
- La elaboración y aplicación de una checklist estructurada permitió validar que los endpoints cumplieran con los requisitos funcionales y de negocio definidos, reduciendo riesgos de omisiones y asegurando cobertura de pruebas.

2. Se fortaleció la robustez del backend mediante pruebas exhaustivas
Las pruebas manuales con Postman, utilizando distintos métodos HTTP (GET, POST, PUT, DELETE) y datos tanto válidos como inválidos, permitieron:
- Verificar códigos de estado HTTP correctos.
- Validar mensajes de error coherentes y descriptivos.
- Confirmar la correcta validación de campos obligatorios.
- Comprobar la integridad y estructura adecuada de las respuestas JSON.

3. Se mejoró la seguridad y control de accesos
- La validación de autenticación y pruebas de accesos no autorizados aseguraron que los mecanismos de seguridad funcionaran correctamente, protegiendo la información y los recursos del sistema.

4. Se validó la lógica de negocio crítica
- La comprobación del manejo de inventario, actualización de stock y modificación de precios confirmó la correcta implementación de reglas de negocio esenciales para una plataforma de comercio electrónico.

5. Se contribuyó activamente a la mejora continua del producto
- La detección y documentación detallada de bugs en Jira, incluyendo pasos de reproducción y evidencias, facilitó la corrección eficiente por parte del equipo de desarrollo y promovió una comunicación clara dentro del equipo Scrum.

6. Se fortalecieron competencias técnicas y metodológicas
- El uso de herramientas como Postman y Jira, junto con la participación en un entorno ágil bajo metodología Scrum, consolidó habilidades en pruebas de API, gestión de incidencias y trabajo colaborativo.


### Conclusión General
El proyecto permitió asegurar la calidad, confiabilidad y seguridad del backend de Urban Grocers antes de su despliegue, minimizando riesgos en producción y contribuyendo directamente a una mejor experiencia para el usuario final. Asimismo, evidenció un enfoque estructurado de pruebas orientado a la detección temprana de defectos y al cumplimiento riguroso de los requerimientos del negocio.
